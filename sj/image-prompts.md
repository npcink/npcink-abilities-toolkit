# Image Prompts

## Shared Series Identity

Use a consistent Npcink developer-tool visual language across future assets:

- Modern open-source developer tooling.
- Clean geometric shapes.
- Deep charcoal or clean off-white base.
- WordPress-like blue accents.
- Green validation accents.
- A small magenta accent used sparingly as the Npcink series signature.
- No official WordPress logo, MCP logo, AI company logos, or trademarked marks.

## Icon Prompt

```text
Create a square WordPress.org plugin icon for "Npcink Abilities Toolkit", one plugin in the Npcink developer-tool series.

This specific plugin represents ability definitions, ability callbacks, reusable ability content packages, and contract specifications for the WordPress Abilities API. It is not the governance core, not an adapter/transport layer, not a cloud connector, and not a model client.

Visual metaphor: a compact set of modular ability blocks connected to a central callback node, with subtle schema brackets, contract lines, and validation marks. The image should feel like a precise developer infrastructure component inside a larger plugin suite.

Series identity: modern open-source developer tooling, clean geometric shapes, consistent Npcink developer-tool style, restrained technical design.

Color direction: deep charcoal or ink base, WordPress-like blue accents, green validation accents, small magenta accent used sparingly as the Npcink series signature.

Style: flat vector, crisp, high contrast, professional, readable at 128x128 and 256x256.

Avoid: robot faces, chat bubbles, cloud product imagery, governance/audit dashboards, approval stamps, network adapter plugs, official WordPress logo, MCP logo, AI company logos, tiny unreadable text, fantasy magic wand imagery.
```

## Banner Prompt

```text
Create a WordPress.org plugin banner for "Npcink Abilities Toolkit", size 1544x500, with a layout that also works when scaled down to 772x250.

This plugin is part of the Npcink developer-tool series. Its role is the ability content package and contract specification layer: ability definitions, ability callbacks, schema normalization, annotations, safe agent-callable WordPress capabilities, and compatibility with hosts that consume the WordPress Abilities API.

It complements the other Magick AI plugins:
- Magick AI Core handles governance, approval, preflight, and audit.
- Magick AI Adapter handles OpenClaw channel adaptation and calls Core plus the Abilities API.
- Magick AI Cloud Addon links to cloud services.
This banner should clearly represent Abilities, not Core, Adapter, or Cloud.

Visual concept: a WordPress ability catalog made of modular capability cards, callback nodes, schema contract brackets, and validation boundaries. Show abilities flowing into host/plugin clients through clean contract lines, not through a cloud or chatbot interface.

Text: "Npcink Abilities Toolkit"
Optional subtitle: "Ability packages and callback contracts for the WordPress Abilities API"

Tone: precise developer infrastructure, safe extensibility, open-source WordPress ecosystem, calm and trustworthy.

Series identity: use a visual language that can later extend to Core, Adapter, and Cloud Addon: same geometry, same base palette, different symbolic focus.

Color direction: deep charcoal or clean off-white base, WordPress-like blue, green validation accents, small magenta/magic accent used sparingly.

Composition: strong readable title area, abstract technical system on the opposite side, generous margins, no critical details near edges, balanced enough for LTR and RTL contexts.

Avoid: robots, chatbot UI, cloud dashboards, approval/audit panels, transport bridges, official WordPress logo, MCP logo, AI company logos, fantasy magic effects, marketing hype, small unreadable code text.
```
